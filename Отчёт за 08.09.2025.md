# Изучение HTTP-запросов для FastAPI

## Что такое HTTP-запросы?

**HTTP (HyperText Transfer Protocol)** — это протокол передачи данных, фундаментальная основа общения в интернете. Проще говоря, это набор правил, по которым клиент (например, браузер) запрашивает данные у сервера, а сервер отвечает на эти запросы.

**HTTP-запрос** — это сообщение, которое клиент отправляет серверу.
*   **Пример:** Когда вводите `youtube.com` в браузере, браузер отправляет HTTP-запрос к серверам YouTube: «Дай мне главную страницу».

## Сфера применения

*    **Веб-сайты:** Открытие любых страниц в браузере.
*    **Мобильные приложения:** Получение и отправка данных на сервер (через HTTPS).
*    **API и микросервисы:** Обмен данными между разными частями программного обеспечения. **FastAPI как раз создан для простого и эффективного построения таких API.**
*    **IoT устройства:** Умные устройства (камеры, датчики) используют HTTP для связи с серверами.

## Основные HTTP-методы (виды запросов)

В FastAPI явно определяется, какой метод обрабатывает endpoint (маршрут).

| Метод     | Описание                                          | Пример использования в FastAPI       |
| :-------- | :------------------------------------------------ | :----------------------------------- |
| **GET**   | Получить данные.                                  | Получить список пользователей.       |
| **POST**  | Создать новый объект, отправить данные.           | Создать нового пользователя.         |
| **PUT**   | Полностью обновить существующий объект.           | Обновить все данные пользователя.    |
| **PATCH** | Частично обновить существующий объект.            | Изменить email пользователя.         |
| **DELETE**| Удалить объект.                                   | Удалить пользователя.                |

## Коды состояния HTTP (Status Codes)

Сервер всегда отвечает кодом, который указывает на результат обработки запроса. В FastAPI вы можете легко возвращать нужные коды состояния.

*   **2xx — Успех (Success) **
    *   `200 OK` — Стандартный ответ для успешных запросов.
    *   `201 Created` — Запрос успешен и был создан новый ресурс (часто для POST).

*   **3xx — Перенаправление (Redirection) **
    *   `301 Moved Permanently` — Ресурс был окончательно перемещен.
    *   `302 Found` — Временное перенаправление.

*   **4xx — Ошибка клиента (Client Error) **
    *   `400 Bad Request` — Сервер не может обработать запрос из-за ошибки клиента.
    *   `401 Unauthorized` — Требуется аутентификация.
    *   `403 Forbidden` — Доступ запрещен (у вас нет прав).
    *   `404 Not Found` — Запрашиваемый ресурс не найден.

*   **5xx — Ошибка сервера (Server Error) **
    *   `500 Internal Server Error` — Общая ошибка сервера.
    *   `502 Bad Gateway` — Проблема с промежуточным сервером.
    *   `503 Service Unavailable` — Сервер перегружен или на обслуживании.

## Заключение

Понимание HTTP — критически важно для работы с любым веб-фреймворком, включая **FastAPI**. Каждое API-приложение будет принимать HTTP-запросы и возвращать HTTP-ответы, поэтому знание методов, статус-кодов и общей механики протокола является основой для разработки эффективных и корректных бэкенд-сервисов.
